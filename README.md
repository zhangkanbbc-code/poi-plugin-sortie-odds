# poi-plugin-sortie-odds（全程胜率）

根据 poi 当前舰队、KCNav 路线与敌编成统计，在本地运行 KC3 Sortie Simulator，估算从当前状态到目标点的：

- 全程 S 胜率
- 全程 A 胜以上概率
- 目标点到达率
- 抵达后的 S / A 胜率
- 每个战斗点的到达率与大破率
- 当前编成走完所选路线的历史证据（带路可达性）

## v0.4 行为

1. 输入海域并选择目标点。
2. 插件从 KCNav 地图数据枚举可达路线；多条路线时由用户选择。
3. 出击后自动读取 `api_req_map/start` / `api_req_map/next` 的 edge ID，锁定实际走过的路线。
4. 到达能动分歧时暂停后续分析；选路完成、收到新 edge 后自动重算。
5. 每个战斗点从 KCNav 获取敌编成及样本权重，在隐藏的本地页面中执行蒙特卡洛模拟。
6. 用舰种构成、舰队类型/编号及电探/运输桶数量查询 KCNav，核对是否存在走完所选路线的历史记录。
7. 读取航海日志（`poi-plugin-akashic-records`）的最新地图点结算，校验已经完成的路线前缀。
8. 读取未卜先知（`poi-plugin-prophet`）的战后舰队状态，把结算后的 HP 作为剩余路线模拟起点。
9. 默认开启自动跟随：进点更新路线，战斗中暂停重算，结算与战后 HP 到齐后自动分析剩余行程。
10. 通常海域只按当前所选路线读取敌编成并保存本地缓存；不会批量抓取 KCNav，缓存更新失败时继续使用已有数据。
11. 所有 KCNav 请求经全局串行队列（1.5~3 秒随机间隔）；收到自动化拒绝自动熔断 6/12/24 小时并持久化，期间只用本地数据。
12. 活动图数据也保存本地缓存（地图 24 小时、敌编成 12 小时）；带路核验结果按「地图+路线+编成特征」缓存。

不会上传舰娘 ID、等级、装备明细或当前状态；带路核验请求只包含海域、edge ID、舰种构成、舰队类型/编号及电探/运输桶数量。

## 当前已计入

- 当前出击舰队、联合舰队
- 当前 HP、油、弹、疲劳
- 装备、改修、熟练度
- 敌编成出现权重
- 沿途损伤、舰载机损失和大破撤退
- 空袭、航空战、夜战点的基本类型
- 目标点默认总是进入夜战（可切换为"未达 A 才进"或"不进"）
- 地图拓扑、实际路线前缀与 KCNav 同类编成历史证据
- 结算后跳过已完成节点，只重算尚未完成的战斗
- 航海日志节点/战果校验与未卜先知战后 HP

## v0.7 已接入

- 道中/决战支援（自动检测 33/34 号支援远征）
- 基地航空队（自动读取出击基地与实时机数；跟随中按玩家实际派遣分配波次）
- 活动难度过滤、血条期数与斩杀/通常形态样本筛选
- 解密条件展示（KCNav 众包）与运输 TP 结算
- 烟幕（发动率内置公式，命中补正为社区检证值）
- 特效/破甲手动倍率（数值按当期社区整理自行填写）
- 目标点阵形（特攻自动选阵）与夜战策略

## 后续需要接入

解密进度游戏 API 不暴露，开路未完成时以游戏内实际带路为准。

## 安装

从 [Releases](https://github.com/zhangkanbbc-code/poi-plugin-sortie-odds/releases) 下载最新 tgz，
在 poi 设置 → 插件 → 安装本地插件包 中选择该文件即可。
建议同时安装 poi-plugin-prophet（未卜先知）与 poi-plugin-akashic-records（航海日志）以启用联动。

## 活动新敌人数据

模拟器数据（`engine/vendor/`）来自 KC3 kancolle-replay，随插件版本固定。
每期活动的新深海舰（尤其陆基类）需等上游收录后同步——运行
`npx tsx scripts/update-vendor.ts` 一键更新到上游最新。遇到
「Unknown ship: XXXX, stats required」报错即表示该敌尚未收录；
插件在此情况下选择明确报错而非用错误模型硬算（缺陆基分类的敌人
会被误算为可被鱼雷命中的水面舰，高估胜率）。

## 开发

```powershell
npm install
npm run check
```

构建后入口文件为根目录的 `index.js`。`engine/vendor/` 是固定版本的 KC3 Sortie Simulator 运行时，详见 `THIRD_PARTY_NOTICES.md`。
